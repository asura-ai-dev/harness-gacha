# Task 5003: 検索とタグ絞り込み

## 概要

カタログ画面にキーワード検索とタグ絞り込み機能を実装する。F-1（カタログ閲覧）の検索・タグ絞り込み要件に対応。

## 詳細

### 1. `src/data/catalog.rs` に検索・フィルタ関数を追加

```rust
/// キーワードで pack を検索する
/// name と summary の部分一致（case-insensitive）でフィルタ
pub fn search_packs<'a>(catalog: &'a [CatalogEntry], query: &str) -> Vec<&'a CatalogEntry> {
    if query.trim().is_empty() {
        return listed_packs(catalog);
    }
    let q = query.to_lowercase();
    catalog.iter()
        .filter(|p| p.status == "listed")
        .filter(|p| {
            p.name.to_lowercase().contains(&q)
            || p.summary.to_lowercase().contains(&q)
            || p.description.as_ref().map_or(false, |d| d.to_lowercase().contains(&q))
        })
        .collect()
}

/// タグで pack をフィルタする
pub fn filter_by_tag<'a>(catalog: &'a [CatalogEntry], tag: &str) -> Vec<&'a CatalogEntry> {
    catalog.iter()
        .filter(|p| p.status == "listed")
        .filter(|p| {
            p.tags.as_ref().map_or(false, |tags| tags.iter().any(|t| t == tag))
        })
        .collect()
}

/// カタログ内の全ユニークタグを抽出して返す（ソート済み）
pub fn all_tags(catalog: &[CatalogEntry]) -> Vec<String> {
    let mut tags: Vec<String> = catalog.iter()
        .filter(|p| p.status == "listed")
        .filter_map(|p| p.tags.as_ref())
        .flatten()
        .cloned()
        .collect();
    tags.sort();
    tags.dedup();
    tags
}
```

### 2. `src/app.rs` の `refresh_filtered_ids` メソッドを完全実装

現在のタブ、検索クエリ、アクティブタグに応じて `filtered_ids` を更新するロジックを実装する。

```rust
pub fn refresh_filtered_ids(&mut self) {
    use crate::data::catalog;

    // ベースとなるリスト（タブに応じてフィルタ）
    let mut packs: Vec<&CatalogEntry> = match self.catalog_state.current_tab {
        CatalogTab::Featured => catalog::featured_packs(&self.catalog),
        CatalogTab::Recent => catalog::recent_packs(&self.catalog),
        CatalogTab::Recommended => catalog::listed_packs(&self.catalog),
    };

    // タグフィルタ
    if let Some(tag) = &self.catalog_state.active_tag {
        let tag_clone = tag.clone();
        packs.retain(|p| {
            p.tags.as_ref().map_or(false, |tags| tags.iter().any(|t| t == &tag_clone))
        });
    }

    // 検索フィルタ
    if !self.search_query.is_empty() {
        let q = self.search_query.to_lowercase();
        packs.retain(|p| {
            p.name.to_lowercase().contains(&q)
            || p.summary.to_lowercase().contains(&q)
        });
    }

    self.catalog_state.filtered_ids = packs.iter().map(|p| p.id.clone()).collect();
    // selected_index がはみ出さないように調整
    if self.catalog_state.selected_index >= self.catalog_state.filtered_ids.len() {
        self.catalog_state.selected_index = self.catalog_state.filtered_ids.len().saturating_sub(1);
    }
}
```

### 3. 検索モードのアクションハンドリング

`src/app.rs` の `handle_catalog_action` に検索関連のアクションを追加する。

```rust
Action::Search => {
    // 検索モードのトグル
    self.search_active = !self.search_active;
    if !self.search_active {
        // 検索モード OFF 時にフィルタを更新
        self.refresh_filtered_ids();
    }
}
Action::SearchInput(c) => {
    if self.search_active {
        self.search_query.push(c);
        self.refresh_filtered_ids();
    }
}
Action::SearchBackspace => {
    if self.search_active {
        self.search_query.pop();
        self.refresh_filtered_ids();
    }
}
```

### 4. タグ絞り込みのアクションハンドリング

`Action::ToggleTag` でタグ選択をサイクルする。カタログ内の全タグを取得し、None -> tag1 -> tag2 -> ... -> None とサイクルさせる。

```rust
Action::ToggleTag => {
    let tags = crate::data::catalog::all_tags(&self.catalog);
    if tags.is_empty() {
        return;
    }
    self.catalog_state.active_tag = match &self.catalog_state.active_tag {
        None => Some(tags[0].clone()),
        Some(current) => {
            if let Some(pos) = tags.iter().position(|t| t == current) {
                if pos + 1 < tags.len() {
                    Some(tags[pos + 1].clone())
                } else {
                    None // 最後のタグの次は None（フィルタ解除）
                }
            } else {
                None
            }
        }
    };
    self.catalog_state.selected_index = 0;
    self.refresh_filtered_ids();
}
```
