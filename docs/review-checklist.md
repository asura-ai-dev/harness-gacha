# Pack Review Checklist

## 目的

このチェックリストは、harness pack をストアに掲載する前に、権利、互換性、安全性、商品表示、運用上の問題を確認するための審査基準である。`manifest.json` と実ファイルを突き合わせながら、掲載可否を判断する。

## 使い方

- 1 pack ごとに 1 回レビューを行う
- 更新版の提出時も再レビューする
- 各項目を `pass`、`fail`、`needs follow-up` のいずれかで記録する
- `fail` が 1 つでもある場合は公開しない
- `needs follow-up` がある場合は creator へ差し戻す

## 1. 提出物の完全性

- `manifest.json` が pack ルートに存在する
- `manifest.json` が JSON として正しく読める
- 必須フィールドがすべて存在する
- `contents` に記載されたファイルが実際に存在する
- install に必要なファイルが欠けていない
- README または商品説明に相当する説明資料がある

## 2. 権利と出品資格

- creator 情報が確認できる
- creator が提出物の権利を持っていると表明している
- 第三者コード、第三者 prompt、第三者テンプレートの流用条件が確認できる
- ライセンス表記が提出物と矛盾していない
- 無断転載やブランド誤認の疑いがない
- OpenAI / Anthropic / 他社との公式提携を誤認させる表現がない

## 3. 商品説明と表示の正確性

- pack 名が誤認を招かない
- summary と description が実内容と一致している
- 対応ツール名と対応 version が明確に書かれている
- install 方法の説明が実際の手順と一致している
- 同梱ファイル一覧が実ファイルと一致している
- 権限表示が実際の挙動と一致している
- 外部通信の説明がある場合、その内容が正確である
- 購入前表示に必要な情報をストアへ出せる状態になっている

## 3.5. 購入前プレビューの情報境界

- 購入前プレビューが pack の価値判断に必要な情報を提供している
- 購入前プレビューが pack をそのまま複製できる粒度になっていない
- prompt 全文、hook 全文、template 完全版、install script 詳細実装を露出していない
- 同梱物の見せ方がカテゴリ、件数、抜粋、要約中心になっている
- サンプル表示が著作権や営業価値を過度に毀損しない

## 4. 互換性

- `targets` に最低 1 件以上の対応ツールがある
- 対応ツールごとに動作前提が説明されている
- 指定 version range が不自然に広すぎない
- 現行サポート対象バージョンで最低限の確認ができている
- OS 依存がある場合、その制約が明記されている
- 追加依存がある場合、その制約が明記されている

## 5. 安全性

- 不審な shell command がない
- 破壊的な filesystem 操作がない
- 破壊的な git 操作がない
- credential、token、秘密情報の収集や送信がない
- 外部通信先が説明されている
- 外部スクリプトや remote payload の動的取得がない、または明示されている
- user 環境を恒久的に変更する処理がある場合、その内容が明示されている
- 不要に広い権限要求がない

## 6. 導入と利用体験

- install 手順が現実的に実行できる
- install 手順に抜け漏れがない
- post-install の追加作業が必要なら説明されている
- 導入後に期待される動作が説明されている
- エラー時の対処や巻き戻し方法が最低限説明されている

## 7. 品質

- ファイル構成が過度に混乱していない
- 命名が一貫している
- コメントや説明が最低限ある
- pack の価値が README や summary から理解できる
- テンプレや hook が空に近い内容ではない
- 明らかに薄い商品や誇大表現がない

## 8. 商用販売適性

- ストアに掲載できる品質に達している
- 無料配布との差別化ポイントがある
- 購入者が購入前に主要リスクを理解できる
- 購入前に見せる情報だけで代替実装できない
- 返金やサポートで揉めやすい曖昧表現がない
- 有料ランダム販売やコンプ要素を前提にしていない

## 9. 運用リスク

- 削除依頼や権利侵害申立てが来たときに説明可能な状態である
- サポート負荷が極端に高そうな設計ではない
- 今後のアップデート方針が不明瞭すぎない
- 単一 creator 依存リスクが高い場合、その前提を把握している

## 10. 公開判断

### 公開可

- 重大な権利問題がない
- 重大な安全性問題がない
- 商品説明と実内容に大きな齟齬がない
- install と利用が現実的に可能である

### 差し戻し

- 説明不足
- metadata 不足
- install 手順不備
- 軽微ではないが修正可能な安全性懸念

### 不掲載

- 権利侵害の疑いが強い
- 危険な挙動がある
- 権限申告が虚偽
- 誤認を誘う販売表現がある
- 商品としての完成度が不足している
