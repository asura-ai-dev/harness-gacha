# TUI UI 設計

## 目的

この文書は、`harness-gacha` の MVP に必要な TUI 画面、画面遷移、購入前プレビューの情報境界を定義する。MVP の TUI は、発見、比較、信頼判断、購入導線、導入案内に集中し、決済入力そのものは Web checkout に委譲する。

## 設計方針

- TUI の中心は「商品詳細」と「導入」である
- discovery は遊び心を持たせてよいが、販売方式にはしない
- 購入前に安心して判断できる情報は見せる
- 購入前に pack を自作できるほどの情報は見せない
- 危険権限と外部通信は常に目立つ位置に出す
- 購入後の導入体験は最短にする

## ビジュアル方針

- ドット絵風のレトロゲーム感をベースにする
- 滑らかな高級感より、硬い輪郭と明快なコントラストを優先する
- 色数は絞り、1画面で使う主役色を増やしすぎない
- 情報量は多くても、視認性はアーケード UI のように保つ
- `danger`、`safe`、`owned` の状態は色で即判別できるようにする

## スタイルガイド

### 全体トーン

- レトロゲームのストア画面
- 16bit から携帯ゲーム機の中間くらいの雰囲気
- 無機質すぎず、少し遊び心がある

### 画面表現

- 枠線ははっきりした単線または二重線を使う
- 角丸より角張った表現を優先する
- 余白は広すぎず、パネルを積む感覚でレイアウトする
- 選択中の行は反転または高コントラスト色で明示する
- 重要ボタンは点滅ではなく色面で強調する

### タイポグラフィ表現

- ビットマップフォントを連想させる等幅寄りの表示に寄せる
- 見出しは大文字や記号枠で少しゲームらしさを出してよい
- 本文は読みやすさを優先し、過度な装飾は避ける

### モーション表現

- Discovery はカプセル開封やカード出現を想起させる短い演出を許容する
- 通常画面はアニメーションを控えめにし、入力応答を優先する
- 点滅の多用は避ける

### アイコンと記号

- 記号、罫線、ブロック文字でゲーム感を出す
- 絵文字は使わない
- `SAFE`、`RISK`、`OWNED`、`NEW` のような短いラベルを活用する

## カラー運用ルール

- 背景色は暗めにして情報色を浮かせる
- 通常テキストは高コントラストを確保する
- アクセント色は 1 画面で 1 から 2 色に抑える
- 成功、警告、危険、購入済みは意味ごとに固定色を使う
- 色だけに依存せず、ラベルやアイコンでも状態を補足する

## 採用パレット

MVP では `案 D: Cherry Cartridge` を採用する。

採用理由:
- レトロゲーム感とストア感のバランスがよい
- 暗め背景でもアクセントが埋もれにくい
- `購入済み`、`危険権限`、`CTA` の色分離がしやすい
- 世界観がありつつ、長時間見ても重すぎない

## 画面別の色の使い方

### ホーム / カタログ

- 背景は暗く保つ
- 選択行に `accent`
- `owned` の pack はラベル色で識別

### Discovery

- `accent` を最も強く使う
- カプセル開封演出に `accent alt` を使ってよい
- ただし価格や購入導線は過度に煽らない

### Pack 詳細

- 本文は `text_primary`
- metadata は `text_secondary`
- 権限サマリは `success`、`warning`、`danger` を混在させる

### 安全性詳細

- `danger` と `warning` を最も明確に出す
- `safe` は緑でも、説明テキストは中立に保つ

### 購入済みライブラリ

- `owned` を明確に使う
- 更新ありは `warning`
- 未導入は `accent` ではなく `text_secondary` 寄りにする

## 画面一覧

### 1. ホーム / カタログ

役割:
- ストアの入口
- pack の一覧閲覧
- discovery 導線の起点

表示内容:
- featured packs
- recent packs
- recommended packs
- タグまたはカテゴリ
- 検索導線
- `ランダムで探す` アクション

主な操作:
- 一覧移動
- 検索
- タグ選択
- pack 詳細へ遷移
- discovery 画面へ遷移

### 2. Discovery 画面

役割:
- 無料で 1 pack をランダム推薦する
- ガチャ風の楽しさを演出する

表示内容:
- 推薦された 1 pack のカード
- short summary
- creator 名
- 対応ツール
- 価格
- 1 行程度の trust summary

主な操作:
- 詳細を見る
- もう一度探す
- ホームに戻る

注意事項:
- 課金しない
- 排出率を売りにしない
- 当たり、ハズレ表現を使わない

### 2.1 ガチャガチャスクリーン方針

Discovery 画面は、TUI 上でガチャガチャを記号的に再現する専用スクリーンとして扱う。

- 実機の写実的再現ではなく、レトロゲーム風の抽象表現にする
- `待機 -> 演出 -> 結果` の 3 状態で構成する
- 演出は 1 から 2 秒程度で終わる短い体験にする
- レバー、カプセル、結果カードなどのモチーフを使ってよい
- 物理シミュレーションではなく、フレーム切り替えと色変化で気持ちよさを出す

### 2.2 ガチャガチャスクリーンの状態

#### 待機状態

- カプセルマシン風のメイン表示
- `PRESS ENTER TO DISCOVER` のような起動文言
- `ランダムで探す` の CTA
- 説明文は `無料でおすすめ pack を 1 つ表示します` 程度に留める

#### 演出状態

- レバー操作を想起させる表示変化
- カプセル排出または落下を想起させるフレーム更新
- 短い色反転、点灯、記号アニメーション
- 長すぎる待ち時間は避ける

#### 結果状態

- 推薦された pack 名
- summary
- creator
- targets
- 価格
- trust summary
- 同梱カテゴリ件数
- `詳細を見る`
- `もう一度探す`

### 2.3 ガチャガチャスクリーンの表現ルール

- `無料で引く` より `ランダムで探す` を優先する
- `当たり`、`ハズレ`、`排出率` は使わない
- 効果音はテキストで補助してよい
- ASCII、罫線、記号、反転表示でゲーム感を出す
- 写実的なマシン描写より、雰囲気のある UI 記号を優先する

### 2.4 Cherry Cartridge での使い方

- 背景は `#151018`
- マシン本体やパネルは `#241A28`
- 枠線は `#5B3A4A`
- 待機中の CTA は `#FF7A59`
- カプセル出現や結果の強調は `#FFD166`
- 購入済みや保有状態は `#6EC5E9`
- 危険権限は `#E15656`

### 3. Pack 詳細

役割:
- 購入判断の中心画面

表示内容:
- pack 名
- creator
- 価格
- summary
- 対応ツール
- 対応バージョン
- 更新日
- 同梱物サマリ
- 必要権限
- 外部通信の有無
- サンプル抜粋
- 購入ボタン

同梱物サマリの例:
- `skills 3件`
- `hooks 2件`
- `templates 4件`

購入前に見せるもの:
- 何ができる pack か
- どのツールに対応するか
- どの程度の権限が必要か
- 外部通信があるか
- どういう種類のファイルが入っているか
- 短いサンプル抜粋

購入前に見せないもの:
- prompt 全文
- hook 全文
- agent instruction 全文
- template 完全版
- install script の詳細実装
- pack 全ファイル一覧の詳細

主な操作:
- 安全性詳細へ移動
- 購入案内へ移動
- ホームへ戻る

### 4. 安全性詳細

役割:
- pack の権限と影響範囲を明示する

表示内容:
- `shell`
- `network`
- `filesystem_read`
- `filesystem_write`
- `git`
- 想定される影響
- 外部接続先の要約
- post-install 時の注意

主な操作:
- 詳細へ戻る
- 購入案内へ進む

### 5. 購入案内

役割:
- 決済前の最終確認
- Web checkout への遷移

表示内容:
- 商品名
- 価格
- creator
- seller 表示
- 返金ポリシー要約
- Web checkout URL
- QR 表示

主な操作:
- Checkout を開く
- URL をコピー
- 戻る

注意事項:
- TUI 内でカード番号を入力させない

### 6. 購入済みライブラリ

役割:
- 所有 pack の管理

表示内容:
- 購入済み pack 一覧
- version
- 更新有無
- install 状態

主な操作:
- install 詳細へ移動
- 再ダウンロード
- リリースノート確認

### 7. インストール詳細

役割:
- 購入後の導入支援

表示内容:
- install method
- install command
- 手動導入手順
- post-install 手順
- rollback または uninstall の注意

主な操作:
- コマンド表示
- README 表示
- ライブラリへ戻る

### 8. アカウント

役割:
- entitlement と購入履歴の確認

表示内容:
- サインイン状態
- 購入履歴
- ユーザー設定

主な操作:
- サインイン
- サインアウト
- 購入履歴確認

### 9. ヘルプ / 法務情報

役割:
- 商取引表示とサポート導線の提供

表示内容:
- 特商法表示へのリンク
- 利用規約
- 返金ポリシー
- 問い合わせ先

## 画面遷移

```text
ホーム / カタログ
  -> Discovery
  -> Pack 詳細
  -> アカウント
  -> ヘルプ / 法務情報

Discovery
  -> Pack 詳細
  -> ホーム / カタログ

Pack 詳細
  -> 安全性詳細
  -> 購入案内
  -> ホーム / カタログ

安全性詳細
  -> Pack 詳細
  -> 購入案内

購入案内
  -> Web checkout
  -> Pack 詳細

購入済みライブラリ
  -> インストール詳細

インストール詳細
  -> 購入済みライブラリ
```

## 商品詳細の情報設計

### 判断材料として必要な情報

- 何ができる pack か
- 誰が作ったか
- いくらか
- 何に対応するか
- どの権限が必要か
- 外部通信があるか
- どのカテゴリのファイルが何件入るか
- 更新されているか

### コピー防止のために隠す情報

- 文章資産の全文
- core hook や agent 設定の完全版
- 完全な template
- 詳細な script 実装
- 再現可能な粒度の内部ロジック

### 推奨するプレビュー形式

- summary
- スクリーンショットまたは出力例
- 数行の抜粋
- カテゴリ別件数表示
- 主要ファイル名の一部のみ

## 表現ガイド

- `無料で引く` より `ランダムで探す` を優先する
- `ガチャ` は世界観としてのみ使い、販売方式としては押し出さない
- `当たり` `ハズレ` `排出率` のような射幸性の強い文言は避ける
- `おすすめ` `発見` `推薦` の語彙を優先する
